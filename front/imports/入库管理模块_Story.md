# 入库管理模块 - 用户故事

> 模块说明: 管理商品入库全流程，支持采购预约入库和直接入库两种渠道
> 优先级: P0 (MVP必需)

---

## 一、入库单管理

### Story 2.1 采购预约入库单接收
**作为** 仓库收货员  
**我希望** 能够接收ERP推送的采购预约入库单  
**以便** 提前知晓到货信息，做好收货准备

**验收标准:**
- [ ] 自动接收ERP采购订单推送，生成待收货状态的入库单
- [ ] 入库单显示采购订单号（PO号）、供应商信息、预计到货日期
- [ ] 入库单明细显示商品SKU、名称、预期数量
- [ ] 可以查看入库单的生命周期状态（待收货→收货中→验收中→待上架→已完成）
- [ ] 支持按采购订单号、供应商、状态筛选查询
- [ ] 界面标识显示"采购预约"标签（蓝色）

**数据来源:**
- ERP采购订单推送 → WMS生成入库单

---

### Story 2.2 直接入库单创建
**作为** 仓库管理员  
**我希望** 能够在WMS直接创建入库单  
**以便** 处理退货入库、调拨入库、盘盈入库等非采购入库场景

**验收标准:**
- [ ] 可以手动创建入库单，选择入库类型
- [ ] 支持入库类型：退货入库/调拨入库/盘盈入库/赠品入库/样品入库/其他入库
- [ ] 直接入库单无采购订单号和供应商信息（退货入库除外）
- [ ] 可以添加入库商品明细（SKU、数量）
- [ ] 可以保存草稿或提交（提交后状态变为待收货）
- [ ] 界面标识显示"直接入库"标签（灰色）

**与采购预约入库的区别:**
| 特性 | 采购预约入库 | 直接入库 |
|------|-------------|----------|
| 来源 | ERP推送 | WMS创建 |
| 采购订单号 | 有 | 无 |
| 供应商信息 | 有 | 无（退货除外）|
| 预计到货日期 | 有 | 无 |
| 标签颜色 | 蓝色 | 灰色 |

---

### Story 2.3 入库单编辑与取消
**作为** 仓库管理员  
**我希望** 能够编辑或取消未完成的入库单  
**以便** 处理信息变更或异常情况

**验收标准:**
- [ ] 待收货状态的入库单可以编辑商品明细
- [ ] 待收货状态的入库单可以取消（需填写取消原因）
- [ ] 已开始收货（收货中及以上状态）的入库单不可编辑
- [ ] 已取消的入库单可以重新激活或复制创建新单

## 二、收货作业

### Story 2.4 收货登记
**作为** 仓库收货员  
**我希望** 能够对到货商品进行收货登记  
**以便** 确认实际到货数量和状态

**验收标准:**
- [ ] 扫描入库单号或采购订单号定位到待收货单据
- [ ] 扫描商品条码/SKU码自动识别商品
- [ ] 录入实际收货数量
- [ ] 支持部分收货（分批到货场景）
- [ ] 收货完成后提交，状态变为"收货中"
- [ ] 全部商品收货完成后状态变为"验收中"

---

### Story 2.4.1 收货数量差异处理
**作为** 仓库收货员  
**我希望** 能够在收货时处理数量差异  
**以便** 准确记录实际到货数量

**场景说明:**
```
收货时发现实际到货数量与送货单/采购订单不符

差异类型:
- 少货: 实际数量 < 预期数量
- 多货: 实际数量 > 预期数量
- 包装差异: 整箱数量与标注不符
```

**验收标准:**
- [ ] 收货时可以录入实际到货数量
- [ ] 系统自动计算差异数量
- [ ] 差异超过阈值时需要确认原因
- [ ] 少货需要拍照记录现状
- [ ] 可以选择是否部分收货
- [ ] 差异记录自动关联入库单

**差异处理选项:**
```
少货处理选项:
├── 确认收货（按实际数量）
├── 拒收差异部分（只收预期数量）
├── 全部拒收（货物与单据差异过大）
└── 暂存待处理（等待供应商确认）

多货处理选项:
├── 确认收货（按实际数量）
├── 拒收多出部分
└── 暂存待处理（等待供应商确认）
```

---

### Story 2.4.2 收货商品异常处理
**作为** 仓库收货员  
**我希望** 能够在收货时发现商品异常  
**以便** 及时记录和处理

**场景说明:**
```
收货时发现商品本身有问题

异常类型:
- 包装破损: 外包装损坏
- 商品损坏: 商品本体损坏
- 错货: 收到的商品与单据不符
- 串货: 同一SKU不同批次/版本混在一起
```

**验收标准:**
- [ ] 收货时可以选择商品异常标记
- [ ] 异常需要选择具体类型（包装破损/商品损坏/错货/串货）
- [ ] 需要拍照上传异常照片（最多5张）
- [ ] 需要录入异常描述
- [ ] 异常商品与正常商品分开处理
- [ ] 可以选择是否拒收异常商品
- [ ] 异常记录自动关联入库单和供应商

---

### Story 2.5 收货确认
**作为** 仓库收货员  
**我希望** 能够确认收货完成  
**以便** 触发后续验收流程

**验收标准:**
- [ ] 收货列表显示所有已收货商品及数量
- [ ] 可以修改收货数量（在验收前）
- [ ] 确认收货后生成验收任务
- [ ] 系统记录收货人、收货时间
- [ ] 收货差异自动通知ERP（采购入库场景）

## 三、验收作业

### Story 2.6 质量验收
**作为** 仓库质检员  
**我希望** 能够对收货商品进行质量验收  
**以便** 确保入库商品质量合格

**验收标准:**
- [ ] 验收列表显示待验收商品
- [ ] 可以逐件或批量验收
- [ ] 验收结果：合格/不合格
- [ ] 不合格需填写原因：破损/过期/规格不符/质量问题/其他
- [ ] 不合格商品可以拍照记录
- [ ] 效期管理商品必须录入生产日期和过期日期
- [ ] 支持部分验收（分批验收场景）
- [ ] 验收完成后状态变为"待上架"

**效期商品验收:**
- 必须录入：生产日期、过期日期
- 系统校验：过期日期 ≥ 当前日期 + 最小保质期
- 临期商品系统自动标记预警

---

### Story 2.7 质检不通过处理
**作为** 仓库质检员  
**我希望** 能够在质检不通过时记录异常  
**以便** 确定问题商品的处理方式

**场景说明:**
```
验收质检时发现商品不符合质量标准

质检结果分类:
- 合格: 通过质检，可上架
- 待定: 需要进一步确认
- 不合格: 不可上架，需要处理
```

**验收标准:**
- [ ] 质检时可以标记商品为"不合格"
- [ ] 需要选择不合格原因：
  - 外观缺陷（划痕/污渍/变形）
  - 功能故障（无法正常使用）
  - 规格不符（尺寸/颜色/型号不符）
  - 过期/临期（效期不符合要求）
  - 包装问题（包装破损/不完整）
  - 其他
- [ ] 需要拍照上传不合格照片
- [ ] 需要录入详细描述
- [ ] 不合格商品与合格商品分开
- [ ] 生成不合格品处理单

---

### Story 2.9 不合格品处理方案
**作为** 仓库主管  
**我希望** 能够处理不合格品  
**以便** 确定最终处理方式

**处理方案:**

| 处理方式 | 适用场景 | 库存处理 | 财务影响 |
|---------|---------|---------|---------|
| 退货供应商 | 供应商责任 | 移到退货区 | 向供应商索赔 |
| 降价销售 | 轻微瑕疵不影响使用 | 移到降价区 | 损失差价 |
| 报损 | 无法销售 | 移到报损区 | 计入损失 |
| 内部使用 | 员工自用/样品 | 内部领用 | 成本内部消化 |
| 销毁 | 过期/有毒有害 | 实物销毁 | 计入损失 |

**验收标准:**
- [ ] 可以查看所有不合格品记录
- [ ] 可以选择处理方式
- [ ] 需要录入处理数量
- [ ] 需要审批的处理方式（金额/数量阈值）需要主管审批
- [ ] 审批通过后自动更新库存
- [ ] 生成财务需要的处理记录

## 四、上架作业

### Story 2.10 智能库位推荐
**作为** 仓库上架员  
**我希望** 系统能够推荐合适的库位  
**以便** 提高上架效率并优化库位利用

**验收标准:**
- [ ] 系统根据商品属性自动推荐库位：
  - 温度要求匹配（常温/冷藏/冷冻）
  - 安全等级匹配（普通/高值/危险品）
  - 库位尺寸限制匹配
  - 同SKU集中存放（如策略允许）
- [ ] 推荐库位按优先级排序（近出库口优先）
- [ ] 可以手动选择其他库位
- [ ] 推荐库位已满时给出提示

**库位分配规则:**
```
IF 商品.温度要求 = "冷藏" THEN
    推荐 冷藏库位
ELSE IF 商品.是否危险品 = true THEN
    推荐 危险品库位
ELSE IF 商品.是否高价值 = true THEN
    推荐 高值库位
ELSE
    推荐 常温标准位
END IF
```

---

### Story 2.11 上架确认
**作为** 仓库上架员  
**我希望** 能够扫描确认商品上架  
**以便** 准确记录商品存放位置

**验收标准:**
- [ ] 扫描商品条码确认商品
- [ ] 扫描库位码确认上架位置
- [ ] 录入实际上架数量
- [ ] 支持批量上架（同一库位多件商品）
- [ ] 上架完成后更新库存记录
- [ ] 全部上架完成后入库单状态变为"已完成"
- [ ] 系统记录上架人、上架时间

---

### Story 2.12 上架异常处理
**作为** 仓库上架员  
**我希望** 能够处理上架过程中的异常  
**以便** 灵活应对实际情况

**验收标准:**
- [ ] 库位已满时可以申请更换库位
- [ ] 商品破损无法上架时可以标记报损
- [ ] 可以标记实际库位与推荐库位不一致的原因
- [ ] 异常处理需要主管审批

## 五、入库查询与追溯

### Story 2.13 入库单查询
**作为** 仓库管理员  
**我希望** 能够查询历史入库记录  
**以便** 进行数据分析和问题追溯

**验收标准:**
- [ ] 支持按入库单号、采购订单号、供应商、时间范围查询
- [ ] 支持按入库类型、状态筛选
- [ ] 列表显示入库单基本信息和状态
- [ ] 可以查看入库单详情（商品明细、操作记录）
- [ ] 支持导出入库数据

---

### Story 2.14 入库生命周期追踪
**作为** 仓库管理员  
**我希望** 能够追踪入库单的全生命周期  
**以便** 了解每个环节的处理情况

**验收标准:**
- [ ] 入库单详情页显示完整生命周期时间轴
- [ ] 显示每个状态的变更时间和操作人
- [ ] 显示各环节的处理时长
- [ ] 超时未处理的任务标红提醒

**生命周期状态:**
```
待收货 → 收货中 → 验收中 → 待上架 → 已完成
   ↓
已取消（异常结束）
```
