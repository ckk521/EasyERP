# 退换货管理模块 - 用户故事

> 模块说明: 管理客户退货、换货、供应商退货、残次品处理等退换货全流程
> 优先级: P1 (MVP需要基础功能，完整功能Phase 2)
> 关联模块: 入库管理（客户退货入库）、出库管理（换货出库、供应商退货）

---

| 关联模块 | 关联方式 | 说明 |
|---------|---------|------|
| **出库管理** | 换货触发新出库 | 换货流程中，新商品发货通过出库管理模块 |
| **出库管理** | 供应商退货出库 | 供应商退货需要通过出库流程发货 |

| **库存管理** | 库位分配 | 质检合格后重新上架的库位分配 |

## 二、退换货单管理

### Story 4.1 退货单接收
**作为** 仓库管理员  
**我希望** 能够接收ERP推送的客户退货单  
**以便** 及时处理客户退货

**验收标准:**
- [ ] 自动接收ERP退货单推送，生成待收货状态的退货单
- [ ] 退货单显示关联的销售订单号、平台订单号
- [ ] 退货单显示客户信息、退货原因、申请时间
- [ ] 退货单明细显示商品SKU、名称、退货数量
- [ ] 支持按销售订单号、平台订单号、状态查询

**数据来源:**
- 电商平台 → ERP销售模块 → WMS退换货模块

---

### Story 4.2 退货单手工创建
**作为** 仓库管理员  
**我希望** 能够手工创建退货单  
**以便** 处理非销售订单关联的退货（如物流拒收、供应商退货等）

**验收标准:**
- [ ] 可以选择退货类型：客户退货/物流拒收/供应商退货/残次品入库
- [ ] 客户退货需要关联销售订单
- [ ] 物流拒收可以选择关联的原出库单
- [ ] 供应商退货需要选择采购订单
- [ ] 可以录入退货商品明细和退货原因
- [ ] 支持上传退货凭证图片

---

### Story 4.3 退货单状态管理
**作为** 仓库管理员  
**我希望** 能够追踪退货单的完整状态  
**以便** 了解退货处理进度

**验收标准:**
- [ ] 退货单状态流转：待收货 → 收货中 → 验收中 → 待处理 → 已完成/已取消
- [ ] 退货单详情页显示完整生命周期时间轴
- [ ] 显示每个状态的变更时间和操作人
- [ ] 超时未处理的退货单标红提醒
- [ ] 支持取消未开始的退货单

**状态说明:**
```
待收货  - 退货单已创建，等待商品到仓
收货中  - 正在进行收货确认
验收中  - 正在进行质量检验
待处理  - 质检完成，等待根据质检结果处理（重新上架/送修/报损）
已完成 - 退货流程全部完成
已取消 - 退货单已取消
```

## 三、收货与验收

### Story 4.4 退货收货确认
**作为** 仓库收货员  
**我希望** 能够确认退货商品已到仓  
**以便** 开始质量检验

**验收标准:**
- [ ] 扫描退货单号获取退货单信息
- [ ] 核对退货商品明细
- [ ] 扫描商品条码或录入SKU编码
- [ ] 录入实收数量
- [ ] 支持部分收货（客户可能分批寄回）
- [ ] 收货确认后状态变为"验收中"
- [ ] 可以查看客户退货的物流信息（如有）

---

### Story 4.5 退货质量检验
**作为** 仓库质检员  
**我希望** 能够对退货商品进行质量检验  
**以便** 确定商品的处理方式

**验收标准:**
- [ ] 显示退货单明细和商品信息
- [ ] 可以录入质检结果：完好可售/轻微瑕疵/可维修/供应商问题/客户损坏/过期/包装破损
- [ ] 可以录入瑕疵原因描述
- [ ] 可以上传商品瑕疵照片（最多5张）
- [ ] 可以设置处理方式：重新上架/送修/报损/退回供应商
- [ ] 质检结果需要质检员确认
- [ ] 质检完成后状态变为"待处理"

**质检结果说明:**
| 质检结果 | 默认处理方式 | 库存去向 | 财务建议 |
|---------|------------|---------|---------|
| 完好可售 | 重新上架 | 正常库位 | 全额退款 |
| 轻微瑕疵 | 重新上架 | 正常库位 | 部分退款/降价处理 |
| 可维修 | 送修 | 维修区 | 暂挂/维修后处理 |
| 供应商问题 | 退回供应商 | 供应商退货区 | 供应商承担 |
| 客户损坏 | 报损 | 报损区 | 不予退款 |
| 过期 | 报损 | 报损区 | 根据政策处理 |
| 包装破损 | 重新上架（重新包装）| 重新包装区 | 可能退款 |

---

## 四、质检结果处理

### Story 4.6 重新上架处理
**作为** 仓库管理员  
**我希望** 能够将质检合格的退货商品重新上架  
**以便** 恢复为可售库存

**验收标准:**
- [ ] 质检结果为"完好可售"或"轻微瑕疵"的商品可以重新上架
- [ ] 系统推荐库位（优先同类商品区域）
- [ ] 可以修改推荐库位
- [ ] 扫描库位码确认上架位置
- [ ] 上架完成后增加可用库存
- [ ] 重新上架的商品标记为"退货商品"（用于追溯）
- [ ] 支持批量上架

---

### Story 4.7 送修处理
**作为** 仓库管理员  
**我希望** 能够将可维修的退货商品送至维修区  
**以便** 维修后重新上架

**验收标准:**
- [ ] 质检结果为"可维修"的商品可以送修
- [ ] 自动分配到维修区区库位
- [ ] 录入维修要求说明
- [ ] 可以指定维修负责人
- [ ] 维修状态可追踪：待维修/维修中/维修完成
- [ ] 维修完成后可以选择重新上架或报损

---

### Story 4.8 报损处理
**作为** 仓库管理员  
**我希望** 能够确认退货商品报损  
**以便** 记录库存损失

**验收标准:**
- [ ] 质检结果为"客户损坏"、"过期"等不可售商品可以报损
- [ ] 报损需要主管审批（金额超过阈值）
- [ ] 录入报损原因和处理方式（销毁/留存/其他）
- [ ] 可以上传报损凭证照片
- [ ] 报损完成后减少库存
- [ ] 生成报损记录用于财务核算
- [ ] 报损商品移动到报损区

---

### Story 4.9 退回供应商处理
**作为** 仓库管理员  
**我希望** 能够将供应商问题商品退回供应商  
**以便** 向供应商追责

**验收标准:**
- [ ] 质检结果为"供应商问题"的商品可以退回供应商
- [ ] 关联原采购订单和供应商信息
- [ ] 生成供应商退货出库单（关联出库管理模块）
- [ ] 出库单类型为"供应商退货"
- [ ] 可以录入供应商反馈和要求
- [ ] 发货后减少库存
- [ ] 记录退货信息用于财务向供应商索赔

## 五、换货流程

- [ ] 出库单通过出库管理模块处理

## 十、KPI指标

| 指标 | 计算方式 | 目标值 |
|------|---------|--------|
| 退货率 | 退货单数 / 销售单数 | < 5% |
| 退货处理时效 | 从收货到完成的平均时长 | < 48小时 |
| 质检通过率 | 可重新上架数量 / 退货总量 | > 60% |
| 供应商问题退货比 | 供应商原因退货 / 总退货 | < 20% |
| 换货成功率 | 完成换货数 / 换货申请数 | > 95% |

## 与出库管理模块的关联

### 关联点1：换货出库
```
退换货管理模块 (Story 4.12)
    ↓ 生成换货出库单
出库管理模块 (出库单类型=换货出库)
    ↓
通过标准出库流程发货
```

### 关联点2：供应商退货出库
```
退换货管理模块 (Story 4.14)
    ↓ 生成供应商退货出库单
出库管理模块 (出库单类型=供应商退货)
    ↓
通过标准出库流程发货退回供应商
```

### 关联点3：物流拒收二次发货
```
退换货管理模块 (Story 4.18)
    ↓ 物流拒收处理后可选择
出库管理模块 (重新发货)
    ↓
通过标准出库流程二次发货
```
