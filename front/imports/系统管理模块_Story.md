# 系统管理模块 - 用户故事

> 模块说明: 管理用户、角色、权限、系统配置等基础功能
> 优先级: P0 (MVP必需)

---

## 一、用户管理

### Story 7.1 用户账号管理
**作为** 系统管理员
**我希望** 能够管理系统用户账号
**以便** 控制谁能访问系统

**验收标准:**
- [ ] 可以创建用户账号
- [ ] 填写基本信息：用户名、姓名、邮箱、手机号
- [ ] 设置初始密码（默认密码：111111）
- [ ] 可以编辑用户信息
- [ ] 可以重置密码
- [ ] 可以设置用户状态：正常/禁用/离职
- [ ] 可以删除账号（软删除）
- [ ] 列表显示所有用户及状态

**用户状态说明：**
| 状态 | 说明 | 能否登录 | 能否操作 |
|------|------|---------|---------|
| 正常 | 正常使用 | ✅ | ✅ |
| 禁用 | 账号被禁用 | ❌ | ❌ |
| 离职 | 员工离职 | ❌ | ❌（历史数据保留）|

---

### Story 7.2 认证机制（JWT）
**作为** 系统
**我希望** 能够实现JWT Token认证
**以便** 安全地验证用户身份

**验收标准:**
- [ ] 使用JWT Token进行身份认证
- [ ] Access Token有效期1小时
- [ ] Refresh Token有效期7天
- [ ] 支持登录/登出
- [ ] Token中包含用户ID、用户名、角色信息

---

### Story 7.3 会话管理
**作为** 系统管理员
**我希望** 能够管理用户会话
**以便** 控制用户登录状态

**验收标准:**
- [ ] 可配置会话超时时间（默认30分钟无操作自动登出）
- [ ] 记录用户最后登录时间和IP地址
- [ ] 管理员可查看当前在线用户
- [ ] 管理员可强制下线指定用户
- [ ] 支持同一账号多设备登录（可配置）

---

### Story 7.4 敏感操作二次验证
**作为** 系统管理员
**我希望** 对敏感操作进行二次验证
**以便** 防止误操作或恶意操作

**验收标准:**
- [ ] 删除用户需要二次确认弹窗
- [ ] 删除角色需要二次确认弹窗
- [ ] 批量操作需要二次确认弹窗
- [ ] 确认时显示操作摘要

**二次验证场景：**
| 操作类型 | 验证方式 | 说明 |
|---------|---------|------|
| 删除用户 | 弹窗确认 | 显示用户名和关联角色 |
| 删除角色 | 弹窗确认 | 显示角色名称和关联用户数 |
| 批量删除 | 弹窗确认 | 显示将要删除的数据条数 |
| 批量审核 | 弹窗确认 | 显示将要审核的数据摘要 |

---

### Story 7.5 API权限控制
**作为** 系统
**我希望** 对后端API进行权限校验
**以便** 防止未授权的API调用

**验收标准:**
- [ ] 每个接口标注需要的权限标识（如 `inbound:order:create`）
- [ ] 网关/拦截器统一校验Token + 权限
- [ ] 无权限时返回403 Forbidden
- [ ] Token无效或过期时返回401 Unauthorized

## 二、角色权限管理

### Story 7.7 角色管理（MVP简化版）
**作为** 系统管理员
**我希望** 能够管理系统角色
**以便** 按角色分配权限

**验收标准:**
- [ ] 预置3个基础角色：
  - **系统管理员**：全部权限
  - **仓库管理员**：本仓库全部业务权限
  - **操作员**：入库/出库/库存操作权限（不含系统配置）
- [ ] 超级管理员（系统内置，不可删除）
- [ ] 角色列表显示角色名称、编码、描述
- [ ] MVP版本不支持自定义创建角色（Phase 2支持）

**MVP预置角色权限矩阵：**

| 功能模块 | 系统管理员 | 仓库管理员 | 操作员 |
|---------|-----------|-----------|--------|
| 系统管理 | ✅ 全部 | ❌ | ❌ |
| 用户管理 | ✅ 全部 | ❌ | ❌ |
| 仓库管理 | ✅ 全部 | ✅ 全部 | ✅ 查看 |
| 基础数据 | ✅ 全部 | ✅ 全部 | ✅ 查看 |
| 入库管理 | ✅ 全部 | ✅ 全部 | ✅ 操作 |
| 出库管理 | ✅ 全部 | ✅ 全部 | ✅ 操作 |
| 库存管理 | ✅ 全部 | ✅ 全部 | ✅ 查看+盘点 |
| 报表管理 | ✅ 全部 | ✅ 全部 | ✅ 查看 |
| 数据权限范围 | 全部仓库 | 本仓库 | 本仓库 |

**数据权限：**
- 系统管理员：可操作所有仓库数据
- 仓库管理员：只能操作所属仓库的数据
- 操作员：只能操作所属仓库的数据

---

### Story 7.8 权限配置（按钮级）
**作为** 系统管理员
**我希望** 能够配置角色按钮权限
**以便** 控制角色的操作范围

**验收标准:**
- [ ] 支持按钮级权限控制：
  - 查看（列表页、详情页）
  - 新增
  - 编辑
  - 删除
  - 导出
  - 审核/审批
- [ ] 权限修改即时生效
- [ ] 按钮根据权限显示/隐藏

**权限标识规范：**
```
{模块}:{实体}:{操作}
示例：
  inbound:order:view     - 入库单查看
  inbound:order:create   - 入库单新增
  inbound:order:edit    - 入库单编辑
  inbound:order:delete  - 入库单删除
  inbound:order:approve - 入库单审核
```
