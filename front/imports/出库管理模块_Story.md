# 出库管理模块 - 用户故事

> 模块说明: 管理商品出库全流程，包括波次分配、拣货、打包、发货等环节
> 优先级: P0 (MVP必需)

---

## 一、出库单管理

### Story 3.1 销售出库单接收
**作为** 仓库拣货员  
**我希望** 能够接收ERP推送的销售出库单  
**以便** 根据订单要求进行出库作业

**验收标准:**
- [ ] 自动接收ERP销售订单推送，生成待分配状态的出库单
- [ ] 出库单显示销售订单号、客户信息、优先级
- [ ] 出库单明细显示商品SKU、名称、应出数量
- [ ] 可以查看出库单的生命周期状态（待分配→已分配→拣货中→待打包→待发货→已发货）
- [ ] 支持按销售订单号、客户、状态、优先级筛选查询

**数据来源:**
- ERP销售订单推送 → WMS生成出库单

---

### Story 3.2 出库单优先级管理
**作为** 仓库主管  
**我希望** 能够为出库单设置优先级  
**以便** 优先处理紧急订单

**验收标准:**
- [ ] 支持优先级：紧急/高/中/低
- [ ] ERP推送时带入优先级，WMS可调整
- [ ] 紧急订单标红显示并置顶
- [ ] 可以批量调整出库单优先级
- [ ] 优先级影响波次分配和拣货顺序

---

### Story 3.3 出库单合并与拆分
**作为** 仓库主管  
**我希望** 能够合并或拆分出库单  
**以便** 优化拣货效率

**验收标准:**
- [ ] 相同客户的多个出库单可以合并拣货

## 二、波次管理

### Story 3.4 波次策略配置
**作为** 仓库主管  
**我希望** 能够配置波次策略  
**以便** 根据业务场景选择最优的波次分组方式

**波次策略类型:**

| 策略类型 | 编码 | 适用场景 | 订单特点 | 拣货方式 |
|---------|------|---------|---------|---------|
| **按时间策略** | TIME | 日常发货 | 固定时段内订单 | 摘果式 |
| **按快递策略** | CARRIER | 快递分拣 | 同物流商订单集中 | 播种式 |
| **按区域策略** | ZONE | 区域配送 | 同区域订单集中 | 摘果式 |
| **按商品策略** | SKU | 批量拣货 | 含相同商品的订单 | 边拣边分 |

**验收标准:**
- [ ] 可以选择波次策略类型
- [ ] 可以设置每种策略的参数
- [ ] 可以设置默认波次策略
- [ ] 不同波次可以使用不同策略

---

### Story 3.5 波次策略算法

#### Story 3.5.1 按时间策略
**作为** 仓库主管  
**我希望** 能够按时间创建波次  
**以便** 按时段批量处理订单

**算法逻辑:**
```
1. 获取时间范围内的待分配出库单列表
2. 过滤排除特定商品/客户/仓库的订单
3. 计算波次统计信息：订单数、SKU数、总件数
4. 检查最大订单数限制，必要时拆分多个波次
5. 生成波次单，关联出库单
```

**验收标准:**
- [ ] 可以设置波次时间窗口：按小时/按上下午/自定义时段
- [ ] 可以设置波次创建时间：手动/定时自动
- [ ] 时间窗口内的所有待分配订单自动纳入波次
- [ ] 可以设置每个波次的最大订单数（防止波次过大）

---

#### Story 3.5.2 按快递策略
**作为** 仓库主管  
**我希望** 能够按物流商创建波次  
**以便** 方便快递分拣和统一点号

**算法逻辑:**
```
1. 获取指定物流商的待分配出库单
2. 如果启用区域分组，按收货区域再分组
3. 每组生成一个波次
4. 检查每组订单数是否超限，必要时拆分
5. 生成波次单，关联出库单
```

**验收标准:**
- [ ] 可以选择单个或多个物流商创建波次
- [ ] 系统自动筛选指定物流商的待分配订单
- [ ] 同一物流商的订单可以按区域再分组
- [ ] 支持设置每个波次的最大订单数

---

#### Story 3.5.3 按区域策略
**作为** 仓库主管  
**我希望** 能够按收货区域创建波次  
**以便** 方便区域配送和路线规划

**算法逻辑:**
```
1. 获取所有待分配出库单
2. 解析每个订单的收货地址，提取区域信息
3. 按区域维度分组
4. 每组订单生成一个波次
5. 检查每组订单数是否超限，必要时拆分
```

**验收标准:**
- [ ] 可以选择区域维度：按国家/按省份/按城市/按邮区
- [ ] 系统自动解析订单收货地址的区域
- [ ] 同一区域的订单生成一个波次

---

#### Story 3.5.4 按商品策略
**作为** 仓库主管  
**我希望** 能够按商品类型创建波次  
**以便** 优化拣货效率（相同商品批量拣）

**算法逻辑:**
```
1. 收集所有待分配出库单的商品明细
2. 统计每个SKU的出现次数
3. 按出现次数降序排序SKU
4. 选择出现次数 >= 阈值的SKU作为"高频SKU"
5. 将包含高频SKU的订单聚类到一个或多个波次
```

**验收标准:**
- [ ] 可以选择商品维度：按分类/按SKU/按周转率
- [ ] 系统自动分析订单中的商品分布
- [ ] 相同商品达到指定数量时生成波次
- [ ] 支持设置最小商品重复数量阈值

---

### Story 3.6 波次创建
**作为** 仓库主管  
**我希望** 能够创建拣货波次  
**以便** 批量处理订单，提高拣货效率

**验收标准:**
- [ ] 可以按条件筛选出库单创建波次：
  - 按时间范围（如今天所有订单）
  - 按客户/区域
  - 按商品品类
  - 按优先级
- [ ] 可以手动选择出库单加入波次
- [ ] 系统自动计算波次内商品总件数、SKU数
- [ ] 波次创建后出库单状态变为"已分配"
- [ ] 支持创建多个波次并行作业

---

### Story 3.7 波次释放
**作为** 仓库主管  
**我希望** 能够释放波次给拣货员  
**以便** 开始拣货作业

**验收标准:**
- [ ] 可以分配波次给指定拣货员或自动分配
- [ ] 释放时系统自动锁定库存
- [ ] 波次释放后状态变为"拣货中"
- [ ] 可以查看波次的拣货进度
- [ ] 可以暂停/恢复波次

---

### Story 3.8 库存预占与锁定
**作为** 系统  
**我希望** 能够在波次释放时锁定库存  
**以便** 确保拣货时有货可拣

**验收标准:**
- [ ] 波次释放时自动检查库存是否充足
- [ ] 库存充足时锁定对应数量（qty_locked增加）
- [ ] 库存不足时给出预警并标记缺货
- [ ] 支持部分分配（有库存的先分配，缺货的标记）
- [ ] 波次取消时自动释放库存锁定

**库存状态:**
```
总数量(qty_total) = 可用数量(qty_available) + 锁定数量(qty_locked)
```

### Story 3.10 拣货策略配置
**作为** 仓库主管  
**我希望** 能够配置拣货策略  
**以便** 根据业务场景选择最优的拣货方式

**拣货策略类型:**

| 策略类型 | 编码 | 作业方式 | 适用场景 | 优点 | 缺点 |
|---------|------|---------|---------|------|------|
| **摘果式** | PICKING | 按订单逐个拣货 | 订单少、商品多 | 准确率高 | 行走距离长 |
| **播种式** | BATCHING | 先批量拣货，再分播 | 订单多、商品集中 | 减少行走 | 需要二次分拣 |
| **边拣边分** | ZONE_PICK | 拣货同时分拣 | 中等规模 | 效率平衡 | 复杂度高 |

**验收标准:**
- [ ] 可以选择拣货策略类型
- [ ] 可以设置每种策略的参数
- [ ] 可以设置默认拣货策略

---

#### Story 3.10.1 摘果式拣货
**作为** 仓库拣货员  
**我希望** 能够使用摘果式拣货方式  
**以便** 按订单逐个拣选商品

**算法逻辑:**
```
1. 按订单生成拣货任务
2. 计算订单内商品的最优拣货路径
3. 按路径顺序引导拣货
4. 拣货完成后提交
```

**验收标准:**
- [ ] 系统按订单生成拣货任务（每个订单一个任务）
- [ ] 每个任务的拣货清单显示：库位→商品→数量
- [ ] 系统计算最优拣货路径
- [ ] 拣货路径考虑库位顺序和行走距离

---

#### Story 3.10.2 播种式拣货
**作为** 仓库拣货员  
**我希望** 能够使用播种式拣货方式  
**以便** 先批量拣货再分播到各订单

**算法逻辑:**
```
1. 按商品汇总批量拣货任务
2. 批量拣货时按总数量拣取
3. 拣货完成后进入分播环节
4. 分播时扫描订单，分配商品到各订单
```

**验收标准:**
- [ ] 系统按商品生成批量拣货任务
- [ ] 批量拣货清单显示：库位→商品→总数量
- [ ] 拣货时需要按总数量拣取
- [ ] 拣货完成后进入分播环节

---

#### Story 3.10.3 边拣边分
**作为** 仓库拣货员  
**我希望** 能够使用边拣边分方式  
**以便** 在拣货过程中直接分拣到各订单

**算法逻辑:**
```
1. 按库区分组生成拣货任务
2. 每个库区的拣货清单显示该库区所有订单的商品
3. 拣货时显示每个订单需要的数量
4. 拣货后直接确认分配到哪个订单
```

**验收标准:**
- [ ] 系统按库区分组生成拣货任务
- [ ] 每个库区的拣货清单显示该库区所有订单的商品
- [ ] 拣货时显示每个订单需要的数量
- [ ] 拣货后直接确认分配到哪个订单

---

### Story 3.11 智能拣货路径
**作为** 仓库拣货员  
**我希望** 系统能够提供最优拣货路径  
**以便** 减少行走距离，提高拣货效率

**验收标准:**
- [ ] 系统根据库位位置计算最优拣货路径
- [ ] 路径策略可选：S型/Z型/最近优先
- [ ] 按路径顺序展示拣货清单
- [ ] 显示当前位置和下一个目标库位
- [ ] 实时计算预计拣货时间

**拣货路径算法（MVP简化版）:**
```
1. 按库区分组（收货区→存储区→拣货区）
2. 同库区内按巷道排序
3. 巷道内按层排序（从上到下或从下到上）
```

---

### Story 3.13 拣货异常处理
**作为** 仓库拣货员  
**我希望** 能够处理拣货过程中的异常  
**以便** 及时解决问题

**异常类型与处理:**

| 异常类型 | 编码 | 原因 | 处理方式 |
|---------|------|------|---------|
| 库位缺货 | OOS | 库位实际库存不足 | 标记缺货→触发库存调整 |
| 商品错放 | MISPLACE | 商品放错库位 | 记录正确库位→调整库存 |
| 商品破损 | DAMAGED | 商品损坏 | 拍照记录→移至残次品区 |
| 拣货错误 | WRONG | 拣错商品/数量 | 撤销操作→重新拣货 |
| 条码无法识别 | UNREAD | 条码损坏 | 手动输入SKU确认 |

**验收标准:**
- [ ] 可以标记各种异常类型
- [ ] 异常需要录入原因描述
- [ ] 需要拍照上传异常现场照片
- [ ] 异常订单标记为"需处理"
- [ ] 异常需要主管审批后继续
- [ ] 异常处理后自动更新库存
- [ ] 异常记录可追溯

**验收标准:**
- [ ] 按日/周/月统计出库单数量
- [ ] 统计各状态订单数量分布
- [ ] 统计拣货准确率、发货及时率
- [ ] 统计各环节平均处理时长
- [ ] 按客户/商品统计出库量
- [ ] 支持图表展示和导出
