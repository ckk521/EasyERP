# SKU二维码生成规范

> 文档说明: 定义商品、仓库、库位的二维码生成规则，支持H5扫码识别
> 版本: 1.0 | 日期: 2026-04-09

---

## 一、二维码内容格式

### 1.1 通用格式规范

所有二维码采用 **JSON格式** 存储，便于解析和扩展：

```json
{
  "type": "类型标识",
  "code": "唯一编码",
  "name": "显示名称",
  "ext": {扩展字段}
}
```

### 1.2 二维码类型定义

| 类型标识 | 含义 | 示例 |
|----------|------|------|
| `SKU` | 商品SKU | 商品条码 |
| `WH` | 仓库 | 仓库编码 |
| `LOC` | 库位/货架 | 库位编码 |

## 二、商品SKU二维码

### 2.1 二维码内容格式

```json
{
  "type": "SKU",
  "code": "TS-W-S-M-White-Vneck",
  "name": "女士T恤-S码-白色-V领",
  "ext": {
    "barcode": "6912345678901",
    "category": "服装-女装-T恤",
    "spec": "S码/白色/V领",
    "unit": "件"
  }
}
```

### 2.2 生成规则

**方式一：系统生成（推荐）**

```javascript
// 生成商品SKU二维码
generateSKUQRCode(product) {
  const data = {
    type: "SKU",
    code: product.sku_code,           // SKU编码
    name: product.name_cn,            // 商品名称
    ext: {
      barcode: product.barcode,       // 国标条码
      category: product.category_path,// 分类路径
      spec: product.spec,             // 规格
      unit: product.unit              // 单位
    }
  };
  return JSON.stringify(data);
}
```

**方式二：兼容现有条码**

如果商品已有国标条码（EAN/UPC），可以直接使用：

```javascript
// 兼容模式：直接扫描国标条码
generateSKUQRCodeSimple(product) {
  // 只返回SKU编码字符串
  return product.sku_code;
}
```

## 四、库位/货架二维码

### 4.1 二维码内容格式

```json
{
  "type": "LOC",
  "code": "A-01-02-03",
  "name": "A区01排02列03层",
  "ext": {
    "warehouse": "WH-VN-001",
    "zone": "A",
    "aisle": "01",
    "shelf": "02",
    "level": "03",
    "type": "标准位",
    "limit": {
      "weight": "50kg",
      "size": "60x40x40cm"
    }
  }
}
```

### 4.2 生成规则

```javascript
// 生成库位二维码
generateLocationQRCode(location) {
  const data = {
    type: "LOC",
    code: location.code,
    name: `${location.zone}区${location.aisle}排${location.shelf}列${location.level}层`,
    ext: {
      warehouse: location.warehouse_code,
      zone: location.zone,
      aisle: location.aisle,
      shelf: location.shelf,
      level: location.level,
      type: location.type,
      limit: {
        weight: location.weight_limit,
        size: `${location.length_limit}x${location.width_limit}x${location.height_limit}cm`
      }
    }
  };
  return JSON.stringify(data);
}
```

### 4.3 库位编码规则

```
格式：A-01-02-03
      │  │  │  │
      │  │  │  └── 层（01-09）
      │  │  └───── 列（01-99）
      │  └──────── 排（01-99）
      └─────────── 库区（A-Z）

示例：
- A-01-02-03 = A库区，第1排，第2列，第3层
- B-05-10-01 = B库区，第5排，第10列，第1层
```

```

│                             │
│      [二维码图片]            │
│                             │
│      A-01-02-03             │
│   A区01排02列03层            │
│                             │
│   限重：50kg                │
│   尺寸：60x40x40cm          │
```

**张贴位置：**
- 货架每层角落
- 地面堆垛区边缘

## 五、二维码生成工具

### 5.1 后端生成API

```javascript
// Node.js + qrcode 库示例
const QRCode = require('qrcode');
const Canvas = require('canvas');

class QRCodeService {
  
  // 生成纯二维码（返回Base64）
  async generateQRCode(data) {
    const qrData = JSON.stringify(data);
    return await QRCode.toDataURL(qrData, {
      width: 300,
      margin: 2,
      color: {
        dark: '#000000',
        light: '#ffffff'
      }
    });
  }
  
  // 生成带标签的二维码图片（返回Buffer）
  async generateLabelQRCode(data, options) {
    const { code, name, type } = data;
    const { width = 400, height = 300 } = options;
    
    // 创建画布
    const canvas = Canvas.createCanvas(width, height);
    const ctx = canvas.getContext('2d');
    
    // 白色背景
    ctx.fillStyle = '#ffffff';
    ctx.fillRect(0, 0, width, height);
    
    // 黑色边框
    ctx.strokeStyle = '#000000';
    ctx.lineWidth = 2;
    ctx.strokeRect(2, 2, width-4, height-4);
    
    // 生成二维码
    const qrData = JSON.stringify(data);
    const qrCanvas = Canvas.createCanvas(200, 200);
    await QRCode.toCanvas(qrCanvas, qrData, { margin: 1 });
    ctx.drawImage(qrCanvas, 20, 20, 180, 180);
    
    // 添加文字
    ctx.fillStyle = '#000000';
    ctx.font = 'bold 20px Arial';
    ctx.fillText(code, 220, 60);
    
    ctx.font = '16px Arial';
    ctx.fillText(name, 220, 100);
    
    ctx.font = '14px Arial';
    ctx.fillText(`类型：${type}`, 220, 140);
    
    return canvas.toBuffer('image/png');
  }
}
```

## 六、扫码解析规范

### 6.1 H5扫码解析

```javascript
// 扫码结果解析
function parseScanResult(scanText) {
  try {
    // 尝试解析JSON格式
    const data = JSON.parse(scanText);
    
    if (data.type && data.code) {
      // 标准格式
      return {
        type: data.type,      // SKU/WH/LOC
        code: data.code,
        name: data.name,
        ext: data.ext || {}
      };
    }
  } catch (e) {
    // 不是JSON格式，按纯文本处理
  }
  
  // 兼容模式：纯文本（SKU编码/条码）
  return {
    type: 'SKU',  // 默认为商品
    code: scanText,
    name: scanText,
    ext: {}
  };
}

// 使用示例
const result = parseScanResult(scanText);
switch (result.type) {
  case 'SKU':
    handleProductScan(result);
    break;
  case 'WH':
    handleWarehouseScan(result);
    break;
  case 'LOC':
    handleLocationScan(result);
    break;
  default:
    showToast('无法识别的二维码');
}
```

### 6.2 扫码场景处理

| 场景 | 扫码内容 | 处理方式 |
|------|----------|----------|
| 入库收货 | 商品二维码 | 显示商品信息，录入数量 |
| 上架 | 库位二维码 → 商品二维码 | 先扫库位，再扫商品，绑定关系 |
| 拣货 | 库位二维码 → 商品二维码 | 确认库位和商品正确性 |
| 盘点 | 库位二维码 | 显示该库位应存商品清单 |
| 切换仓库 | 仓库二维码 | 切换当前作业仓库 |

## 八、注意事项

### 8.1 二维码容量限制

- QR码最大容量：2953字节（数字）/ 1817字节（字母）
- 建议JSON数据不超过 **500字节**，确保快速识别
- 大数据量时，二维码图案复杂，扫描难度增加

### 8.2 容错等级

```javascript
// 生成时设置容错等级
QRCode.toCanvas(canvas, data, {
  errorCorrectionLevel: 'M',  // L/M/Q/H
  // L: 7% 容错
  // M: 15% 容错（推荐）
  // Q: 25% 容错
  // H: 30% 容错
});
```

### 8.3 打印质量要求

- 打印分辨率：最低300DPI
- 标签材质：防水、防撕、耐磨损
- 张贴位置：平整、无遮挡、光线充足
- 尺寸：最小30x30mm（手机扫描），推荐50x50mm以上
