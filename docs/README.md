## 命名规范

### 需求文档
- 每个功能模块创建独立文件夹
- 子功能用 3 位数字编号（001, 002, ...）
- 文件夹内包含 `需求/` 和 `测试用例/` 两个子文件夹

### 测试用例
- 文件名前缀与需求文档编号对应
- 如 `01_用户管理测试用例.md`, `02_角色权限测试用例.md`

## 文档维护

1. **需求变更** - 更新对应模块的需求文件夹
2. **新增功能** - 在对应模块下创建新的需求文件夹
3. **测试更新** - 同时更新对应模块的测试用例
4. **进度更新** - 更新 `产品规划.md` 中的进度表格
