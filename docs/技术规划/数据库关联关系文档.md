## 二、核心关联链路

### 2.1 入库流程数据链路

```
ERP采购订单
    │
    │ 1:1 (erp_purchase_order_id)
    ▼
WMS入库单 (inbound_order)
    │
    │ 1:N (inbound_order_id)
    ▼
WMS入库单明细 (inbound_order_item)
    │
    │ N:1 (product_id)
    ▼
商品 (product)
    │
    │ 1:N (通过入库上架生成)
    ▼
库存记录 (inventory)
    │
    │ N:1 (location_id)
    ▼
库位 (location)
```

### 2.2 出库流程数据链路

```
ERP销售订单
    │
    │ 1:1 (erp_sales_order_id)
    ▼
WMS出库单 (outbound_order)
    │
    │ 1:N (outbound_order_id)
    ▼
WMS出库单明细 (outbound_order_item)
    │
    │ N:1 (product_id)
    ▼
库存分配 (分配时锁定库存)
    │
    │ 更新 (inventory.qty_locked)
    ▼
库存记录 (inventory)
```

### 2.3 库存核心关联

```
                    ┌─────────────┐
                    │   仓库      │
                    │  warehouse  │
                    └──────┬──────┘
                           │ 1:N
                           ▼
                    ┌─────────────┐
                    │   库区      │
                    │    zone     │
                    └──────┬──────┘
                           │ 1:N
                           ▼
┌──────────┐      ┌─────────────┐      ┌──────────┐
│   商品   │◄─────┤   库位      ├─────►│  库存    │
│ product  │ 1:N  │  location   │ 1:N  │inventory │
└────┬─────┘      └─────────────┘      └────┬─────┘
     │                                      │
     │ 1:N                                  │ N:1
     ▼                                      ▼
┌──────────┐                         ┌──────────┐
│  商品SKU │                         │  入库单  │
│product_  │                         │ inbound  │
│  sku     │                         │  _order  │
└──────────┘                         └──────────┘
```

## 四、冗余字段策略

### 4.1 冗余原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **查询频繁** | 跨表查询频繁字段冗余 | inbound_order.supplier_name |
| **稳定性高** | 不常变更的字段可以冗余 | product.sku_code 在 inventory |
| **关键路径** | 核心业务流程关键字段冗余 | outbound_order_item.sku_code |

### 4.2 冗余字段清单

| 表名 | 冗余字段 | 来源表 | 来源字段 | 同步策略 |
|------|----------|--------|----------|----------|
| inbound_order | supplier_name | supplier | name | 创建时复制 |
| inbound_order_item | sku_code | product | sku_code | 创建时复制 |
| inbound_order_item | product_name | product | name_cn | 创建时复制 |
| outbound_order | platform_order_no | erp_sales | platform_order_no | 创建时复制 |
| outbound_order_item | sku_code | product | sku_code | 创建时复制 |
| outbound_order_item | product_name | product | name_cn | 创建时复制 |
| inventory | sku_code | product | sku_code | 创建时复制 |
| inventory | warehouse_id | location | warehouse_id | 关联获取 |

### 4.3 冗余数据一致性保障

```sql
-- 示例：触发器保持冗余字段同步（可选）
-- 供应商名称变更时，更新相关入库单
DELIMITER //
CREATE TRIGGER trg_supplier_name_update
AFTER UPDATE ON supplier
FOR EACH ROW
BEGIN
    IF OLD.name != NEW.name THEN
        UPDATE inbound_order 
        SET supplier_name = NEW.name 
        WHERE supplier_id = NEW.id;
    END IF;
END//
DELIMITER ;
```

## 六、系统管理模块数据库设计

> 本节定义系统管理模块（用户、角色、权限、配置、日志、消息通知）的表结构

### 6.1 用户管理

#### sys_user（用户表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR(200) | NOT NULL | 密码（BCrypt加密） |
| real_name | VARCHAR(100) | NOT NULL | 姓名 |
| email | VARCHAR(100) | - | 邮箱 |
| phone | VARCHAR(20) | - | 手机号 |
| avatar | VARCHAR(500) | - | 头像URL |
| status | TINYINT | NOT NULL, DEFAULT 1 | 状态：0-禁用 1-正常 2-离职 |
| warehouse_id | BIGINT | - | 所属仓库ID（仓库管理员/操作员必填） |
| last_login_at | DATETIME | - | 最后登录时间 |
| last_login_ip | VARCHAR(50) | - | 最后登录IP |
| password_changed_at | DATETIME | - | 密码修改时间 |
| password_expired | TINYINT | DEFAULT 0 | 密码是否过期：0-否 1-是 |
| must_change_pwd | TINYINT | DEFAULT 0 | 是否必须修改密码 |
| fail_login_count | INT | DEFAULT 0 | 连续登录失败次数 |
| locked_until | DATETIME | - | 锁定截止时间 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |
| deleted_at | DATETIME | - | 删除时间（软删除） |

```sql
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键',
    username VARCHAR(50) NOT NULL COMMENT '用户名',
    password VARCHAR(200) NOT NULL COMMENT '密码（BCrypt加密）',
    real_name VARCHAR(100) NOT NULL COMMENT '姓名',
    email VARCHAR(100) COMMENT '邮箱',
    phone VARCHAR(20) COMMENT '手机号',
    avatar VARCHAR(500) COMMENT '头像URL',
    status TINYINT NOT NULL DEFAULT 1 COMMENT '状态：0-禁用 1-正常 2-离职',
```
