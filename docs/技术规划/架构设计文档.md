## 一、架构决策记录

| 决策项 | 决策内容 | 原因 |
|--------|----------|------|
| 平台对接 | Phase 1不做 | 先验证WMS核心功能 |
| PDA端 | H5实现，Phase 1不做 | MVP阶段暂缓，Phase 2实现 |
| 库存模型 | ERP+WMS统一库存 | 简化架构，避免同步问题 |
| 部署方式 | 单体应用+多租户 | 快速开发，支持多仓库 |
| 多租户 | 不使用 | 无租户概念，仓库自主配置 |
| 高价值商品 | 三级分类 | 普通/高价值/严格管控 |

## 五、统一库存模型设计

### 5.1 核心库存表

```sql
-- 库存主表
-- 注意：分表策略由DBA在数据库层面实现，应用层透明
CREATE TABLE inventory (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    warehouse_id BIGINT NOT NULL COMMENT '仓库ID',
    location_id BIGINT NOT NULL COMMENT '库位ID',
    product_id BIGINT NOT NULL COMMENT '商品ID（真实外键）',
    sku_code VARCHAR(50) NOT NULL COMMENT 'SKU编码（冗余，用于快速查询）',
    batch_no VARCHAR(50) DEFAULT '' COMMENT '批次号',

    -- 库存数量
    qty_total INT DEFAULT 0 COMMENT '总数量',
    qty_available INT DEFAULT 0 COMMENT '可用数量',
    qty_locked INT DEFAULT 0 COMMENT '锁定数量(已分配未出库)',

    -- 效期管理
    production_date DATE COMMENT '生产日期',
    expiry_date DATE COMMENT '过期日期',

    -- 乐观锁（用于并发控制）
    version INT DEFAULT 0 COMMENT '版本号',

    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    -- 外键约束（product_id真实关联，sku_code仅冗余存储用于显示）
    FOREIGN KEY (product_id) REFERENCES product(id),
    FOREIGN KEY (warehouse_id) REFERENCES warehouse(id),
    FOREIGN KEY (location_id) REFERENCES location(id),

    -- 索引
    UNIQUE KEY uk_inventory (warehouse_id, location_id, product_id, batch_no),
    KEY idx_sku (sku_code),
    KEY idx_product (product_id),
    KEY idx_location (location_id),
    KEY idx_expiry (expiry_date)
) ENGINE=InnoDB COMMENT='库存表';

-- 为什么要用 product_id 而不是仅用 sku_code？
-- 1. product_id 是真实外键，数据一致性有保障
-- 2. sku_code 可能变更（如商品编码规则调整），但 product_id 不变
-- 3. 库存表通过 product_id 关联可确保溯源准确
-- 4. sku_code 作为冗余字段保留，用于快速查询和显示
```

### 5.2 流水号生成策略

```sql
-- 使用数据库自增序列实现流水号
-- 每个业务表有独立的序列号段，避免冲突

-- 示例：入库单号流水表
CREATE TABLE seq_inbound_order (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    prefix VARCHAR(10) DEFAULT 'RK' COMMENT '前缀',
    date_str VARCHAR(8) DEFAULT '' COMMENT '日期字符串',
    current_value INT DEFAULT 0 COMMENT '当前值',
    PRIMARY KEY (id),
    UNIQUE KEY uk_date (date_str)
) ENGINE=InnoDB COMMENT='入库单号序列';

-- 获取下一个入库单号的存储过程
DELIMITER //
CREATE PROCEDURE sp_get_next_inbound_no(OUT p_order_no VARCHAR(30))
BEGIN
    DECLARE v_date_str VARCHAR(8);
    DECLARE v_seq INT;

    SET v_date_str = DATE_FORMAT(NOW(), '%Y%m%d');

    -- 尝试更新现有记录
    UPDATE seq_inbound_order
    SET current_value = current_value + 1
    WHERE prefix = 'RK' AND date_str = v_date_str;

    -- 如果没有记录，则插入新记录
    IF ROW_COUNT() = 0 THEN
        INSERT INTO seq_inbound_order (prefix, date_str, current_value)
        VALUES ('RK', v_date_str, 1);
        SET v_seq = 1;
    ELSE
        SELECT current_value INTO v_seq
        FROM seq_inbound_order
        WHERE prefix = 'RK' AND date_str = v_date_str;
    END IF;

    SET p_order_no = CONCAT('RK', v_date_str, LPAD(v_seq, 4, '0'));
END//
DELIMITER ;

-- 调用示例
-- CALL sp_get_next_inbound_no(@order_no);
-- SELECT @order_no;  -- 输出：RK202604100001

-- 优势：
-- 1. 数据库自增，高并发下不会重复
-- 2. 按日期分区，不同日期独立计数
-- 3. 存储过程保证原子性
-- 4. 简单易维护
```

```sql
-- 库存流水表
CREATE TABLE inventory_log (
```

### 5.4 库存变动规则

```
┌─────────────────────────────────────────────────────────────┐
│                     库存变动规则                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 入库上架                                                │
│     qty_total + N                                           │
│     qty_available + N                                       │
│                                                             │
│  2. 波次分配 (预占)                                         │
│     qty_available - N                                       │
│     qty_locked + N                                         │
│                                                             │
│  3. 发货确认 (扣减)                                         │
│     qty_total - N                                           │
│     qty_locked - N                                          │
│                                                             │
│  4. 分配取消 (释放)                                         │
│     qty_available + N                                       │
│     qty_locked - N                                          │
│                                                             │
│  5. 盘点调整                                                │
│     根据实盘数量调整 qty_total 和 qty_available             │
│                                                             │
└─────────────────────────────────────────────────────────────┘

校验规则:
- qty_total = qty_available + qty_locked
- qty_available >= 0
- qty_locked >= 0
- qty_total >= 0
```

## 六、RESTful API 设计规范

### 6.1 URL 规范

```
# URL 格式
GET/POST/PUT/DELETE  http://api.easyerp.com/wms/{module}/{entity}

# 模块前缀
/wms/inbound    - 入库模块
/wms/outbound   - 出库模块
/wms/inventory  - 库存模块
/wms/product    - 商品模块
/wms/warehouse  - 仓库模块

GET    /wms/inbound/orders           # 查询入库单列表

PUT    /wms/inbound/orders/{id}      # 更新入库单
DELETE /wms/inbound/orders/{id}      # 删除入库单
```

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| 200 | 成功 | - |

### 10.2 索引设计

```sql
-- 库存查询优化
CREATE INDEX idx_inventory_query ON inventory(warehouse_id, sku_code, qty_available);

-- 效期预警查询
CREATE INDEX idx_expiry_warning ON inventory(expiry_date, qty_available)
WHERE qty_available > 0;

-- 库位查询
CREATE INDEX idx_location_inventory ON inventory(location_id, qty_total);
```
