## 一、系统架构定位

### 1.1 系统整体架构

本系统定位为 **ERP + 仓储管理一体化解决方案**，覆盖企业全业务流程：

### 1.2 核心业务流程

#### 采购流程（入库）

```
供应商 → 采购订单(ERP) → 入库单(ERP→WMS) → 入库作业 → 库存增加

详细步骤：
1. 【ERP采购模块】创建采购订单 → 审批 → 下达给供应商
2. 供应商发货 → 货物到仓
3. 【WMS模块】收货 → 验收 → 上架 → 入库确认
4. 【ERP系统】更新库存账、生成应付账款
```

#### 销售流程（出库）

```
电商平台 → 销售订单(ERP) → 出库单(ERP→WMS) → 出库作业 → 发货 → 库存减少

详细步骤：
1. 【ERP销售模块】同步平台订单 → 创建销售订单 → 审批 → 生成出库单
2. 【WMS模块】接收出库单 → 分配 → 拣货 → 打包 → 发货
3. 【ERP系统】更新库存账、生成应收账款
4. 【平台对接】发货状态回传电商平台
```

### 1.3 ERP vs WMS 功能边界

| 功能模块 | ERP负责（业务管理） | WMS负责（作业执行） |
|---------|-------------------|-------------------|
| **采购** | 供应商管理、采购订单、价格谈判、付款 | 入库执行、上架、入库确认 |
| **销售** | 客户管理、销售订单、定价策略、收款 | 拣货、打包、发货、出库确认 |
| **库存** | 库存账务、成本核算、库存计划、补货建议 | 实物管理、库位管理、盘点 |
| **财务** | 应收应付、发票、成本核算、报表 | 无 |
| **平台对接** | 订单同步、商品上架、库存同步 | 无 |

### 1.4 与电商平台的关系

| 方向 | 数据流 | 说明 |
|------|-------|------|
| **平台 → ERP** | 销售订单 | 从Shopee/Lazada/TikTok同步订单到销售模块 |
| **ERP → 平台** | 商品上架 | 推送商品信息到各平台 |
| **ERP → 平台** | 库存同步 | 实时同步库存数量到各平台 |
| **ERP → 平台** | 发货状态 | 发货后回传物流信息 |

### 1.5 与传统WMS的区别

| 对比项 | 传统WMS | 本系统（ERP+WMS） |
|-------|--------|------------------|
| 订单来源 | 直接对接平台或手工录入 | 从ERP销售模块接收出库单 |
| 库存管理 | 实物库存 | 账务库存 + 实物库存 |
| 财务集成 | 无 | 完整应收应付、成本核算 |
| 采购管理 | 无 | 完整采购流程管理 |
| 平台对接 | 直接对接 | 通过ERP统一对接 |

### 6.2 统一数据模型

#### 商品模型

```typescript
interface UnifiedProduct {
  sku: string                    // SKU编码
  name: string                   // 商品名称
  description: string            // 商品描述
  category: string               // 分类
  price: number                  // 价格
  cost: number                   // 成本
  weight: number                 // 重量(kg)
  dimensions: {                  // 尺寸
    length: number
    width: number
    height: number
  }
  images: string[]               // 图片URL列表
  stock: number                  // 库存数量
```
