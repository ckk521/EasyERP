# 商品分类体系设计

> 基于东南亚跨境电商市场调研，为WMS系统设计的商品分类体系

---

## 一、分类设计原则

1. **覆盖主流品类** - 覆盖东南亚电商及出口主要商品
2. **仓储特性区分** - 不同品类有不同的存储、拣货、打包要求
3. **SKU管理友好** - 支持多规格（尺码、颜色、款式）管理
4. **平台映射便捷** - 便于与Shopee/Lazada/TikTok分类对接

## 四、SKU编码规则

### 4.1 SKU编码结构

```
[分类码]-[品牌码]-[属性码]-[序列号]
   │        │        │        │
   │        │        │        └── 4位数字，区分同款不同批次
   │        │        └── 根据品类不同，包含尺码/颜色/规格等
   │        └── 2-4位字母，品牌缩写
   └── 4位数字，对应分类编码
```

## 六、数据库设计建议

### 6.1 分类表结构

```sql
-- 一级分类表
CREATE TABLE category_l1 (
    id VARCHAR(4) PRIMARY KEY,        -- '01'
    name VARCHAR(50) NOT NULL,        -- '电子数码'
    market_share DECIMAL(5,2),        -- 市场占比
    storage_requirement TEXT,         -- 仓储要求
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 二级分类表
CREATE TABLE category_l2 (
    id VARCHAR(4) PRIMARY KEY,        -- '01'
    l1_id VARCHAR(4) NOT NULL,        -- 关联一级分类 '01'
    name VARCHAR(50) NOT NULL,        -- '手机配件'
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (l1_id) REFERENCES category_l1(id)
);

-- 三级分类表
CREATE TABLE category_l3 (
    id VARCHAR(4) PRIMARY KEY,        -- '01'
    l2_id VARCHAR(8) NOT NULL,        -- 关联二级分类 '0101'
    name VARCHAR(50) NOT NULL,        -- '手机壳'
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (l2_id) REFERENCES category_l2(CONCAT(l1_id, id))
);
```

### 6.2 SKU属性表

```sql
-- SKU属性定义表（按分类配置）
CREATE TABLE sku_attribute_def (
    id SERIAL PRIMARY KEY,
    category_id VARCHAR(8) NOT NULL,   -- 分类编码
    attribute_name VARCHAR(50) NOT NULL, -- 属性名：颜色/尺码/材质
    attribute_code VARCHAR(10) NOT NULL, -- 属性代码：color/size/material
    is_required BOOLEAN DEFAULT true,
    sort_order INT DEFAULT 0
);

-- SKU属性值表
CREATE TABLE sku_attribute_value (
    id SERIAL PRIMARY KEY,
    attribute_id INT NOT NULL,
    value_name VARCHAR(50) NOT NULL,   -- 值名称：黑色
    value_code VARCHAR(10) NOT NULL,   -- 值代码：BK
    sort_order INT DEFAULT 0,
    FOREIGN KEY (attribute_id) REFERENCES sku_attribute_def(id)
);
```

## 七、MVP阶段分类范围

### 7.1 建议优先支持的分类

基于市场调研的中国跨境卖家主要品类分布，MVP阶段建议优先支持：

| 优先级 | 分类 | 理由 |
|--------|------|------|
| **P0** | 电子数码 | 市场占比最高(25-30%)，SKU标准化程度高 |
| **P0** | 服装服饰 | 市场占比高(20-25%)，测试SKU多规格管理 |
| **P0** | 鞋类箱包 | 市场占比高(10-15%)，测试尺码管理 |
| **P1** | 美妆个护 | 市场占比高(8-12%)，测试效期管理 |
| **P1** | 家居生活 | 市场占比高(10-15%)，测试大件管理 |
| **P2** | 其他分类 | 根据客户需求逐步扩展 |

### 7.2 MVP分类数据准备

MVP阶段预置以下分类数据：
- 10个一级分类
- 60个二级分类
- 200个三级分类
- 50个常见品牌
- 20种颜色属性值
- 15个尺码属性值
