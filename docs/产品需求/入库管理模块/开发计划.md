## 一、核心设计原则

### 1.1 数据关联原则

```
所有业务操作必须有数据源，不能凭空创建:

入库单 ──产生──▶ 入库明细 ──产生──▶ 库存记录
   │                │
   │                └──验收不合格──▶ 退货记录
   │                                    │
   │                                    └──关联原入库明细
   │
   └──产生──▶ 库存事务记录

示例:
入库单IN001 ──包含──▶ SKU-001 x 100
                    ├── 验收合格 80件 ──上架──▶ 库存记录
                    └── 验收不合格 20件 ──退货──▶ 退货记录(关联IN001)
```

### 1.2 数据追溯链

```
查询某个商品的完整链路:

SKU-001 库存 80件
├── 来源: 入库单 IN202604210001
├── 批次: IN202604210001-0001
├── 供应商: 深圳电子科技
├── 送货批次: SZ-0421-AM
├── 入库时间: 2026-04-21 14:30
├── 验收人: 张三
├── 上架人: 李四
└── 库位: ZN-A-01-02-03

退货记录 20件
├── 来源: 入库单 IN202604210001 (同一批货)
├── 退货原因: 包装破损
├── 退货时间: 2026-04-21 15:00
└── 处理方式: 退货供应商
```

### 2.1 完整流程图


### 2.2 退货场景说明

```
退货产生的场景:

场景1: 收货时发现商品异常
├── 包装破损 → 拒收 → 直接退货
├── 商品损坏 → 拒收 → 直接退货
├── 错货(商品不对) → 拒收 → 直接退货
└── 数量差异(少货) → 记录差异 → 通知供应商

场景2: 验收时发现不合格
├── 外观缺陷 → 不合格 → 选择处理方式
├── 功能故障 → 不合格 → 选择处理方式
├── 规格不符 → 不合格 → 选择处理方式
└── 效期问题 → 不合格 → 选择处理方式

不合格品处理方式:
┌─────────────┬──────────────────────────────────────────┐
│ 处理方式     │ 说明                                      │
├─────────────┼──────────────────────────────────────────┤
│ 退货供应商   │ 移到退货区，生成退货记录，通知供应商取货    │
│ 降价销售     │ 移到降价区，记录降价原因                    │
│ 报损        │ 移到报损区，计入损失                        │
│ 内部使用     │ 内部领用，成本内部消化                      │
│ 销毁        │ 过期/有毒有害，实物销毁，记录销毁证明        │
└─────────────┴──────────────────────────────────────────┘
```

## 三、数据表设计

### 3.1 入库单主表 (wms_inbound_order)

```sql
CREATE TABLE wms_inbound_order (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    order_no VARCHAR(50) NOT NULL COMMENT '入库单号 IN202604210001',
    delivery_batch_no VARCHAR(50) COMMENT '送货批次号 SZ-0421-AM',
    order_type TINYINT NOT NULL COMMENT '入库类型: 1采购 2退货 3调拨 4赠品 5其他',
    po_no VARCHAR(50) COMMENT '采购订单号(采购入库)',
    
    -- 供应商信息
    supplier_id BIGINT COMMENT '供应商ID',
    supplier_code VARCHAR(50) COMMENT '供应商编码',
    supplier_name VARCHAR(100) COMMENT '供应商名称',
    
    -- 仓库信息
    warehouse_id BIGINT NOT NULL COMMENT '仓库ID',
    warehouse_code VARCHAR(50) COMMENT '仓库编码',
    warehouse_name VARCHAR(100) COMMENT '仓库名称',
    
    -- 日期信息
    expected_date DATE COMMENT '预计到货日期',
    actual_arrival_date DATE COMMENT '实际到货日期',
    
    -- 状态与进度
    status TINYINT DEFAULT 0 COMMENT '状态: 0待收货 1收货中 2验收中 3待上架 4已完成 9已取消',
    progress_receive INT DEFAULT 0 COMMENT '收货进度百分比',
    progress_inspect INT DEFAULT 0 COMMENT '验收进度百分比',
    progress_putaway INT DEFAULT 0 COMMENT '上架进度百分比',
    
    -- 数量统计
    total_expected_qty INT DEFAULT 0 COMMENT '总预期数量',
    total_received_qty INT DEFAULT 0 COMMENT '总收货数量',
    total_qualified_qty INT DEFAULT 0 COMMENT '总合格数量',
    total_rejected_qty INT DEFAULT 0 COMMENT '总不合格数量',
    total_putaway_qty INT DEFAULT 0 COMMENT '总上架数量',
    total_return_qty INT DEFAULT 0 COMMENT '总退货数量',
    
    -- 其他
    remark VARCHAR(500) COMMENT '备注',
    create_user BIGINT COMMENT '创建人',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_user BIGINT,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    complete_time DATETIME COMMENT '完成时间',
    
    UNIQUE KEY uk_order_no (order_no),
    KEY idx_supplier (supplier_id),
    KEY idx_delivery_batch (delivery_batch_no),
    KEY idx_status (status),
    KEY idx_create_time (create_time)
) COMMENT '入库单主表';
```

## 五、数据关联约束

### 5.1 强制关联规则

```
操作              必须关联的数据源
─────────────────────────────────────────────────────────
收货              入库单(已创建)
验收              入库明细(已收货)
上架              入库明细(已验收合格)
退货              入库明细(验收不合格)
库存变动          上架记录/退货记录
调拨              库存记录(已存在)
出库              库存记录(已存在)
```

### 5.2 数据完整性校验

```
入库单完成时校验:
总预期数量 = 总收货数量 + 收货差异数量
总收货数量 = 总合格数量 + 总不合格数量
总合格数量 = 总上架数量 + 待上架数量
总不合格数量 = 总退货数量 + 其他处理数量

库存校验:
库存数量 = SUM(上架记录.上架数量) - SUM(出库记录.出库数量)
```

```

```
