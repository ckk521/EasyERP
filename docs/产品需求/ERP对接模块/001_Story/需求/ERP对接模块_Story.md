# ERP对接模块 - 用户故事

> 模块说明: 实现WMS与ERP系统的数据对接，包括采购订单、销售订单、库存同步、平台对接等
> 优先级: P0 (MVP必需)

---

## 一、采购订单对接

### Story 5.1 采购订单接收
**作为** WMS系统  
**我希望** 能够接收ERP推送的采购订单  
**以便** 生成入库单进行收货准备

**验收标准:**
- [ ] 通过API接收ERP采购订单数据
- [ ] 接收字段包括：PO号、供应商、商品明细、数量、预计到货日期
- [ ] 自动校验商品SKU是否存在
- [ ] 不存在的SKU标记异常并通知
- [ ] 生成待收货状态的入库单
- [ ] 记录对接日志

**对接字段映射:**
| ERP字段 | WMS字段 | 说明 |
|---------|---------|------|
| purchase_order_no | erp_purchase_order_id | 采购订单号 |
| supplier_code | supplier_id | 供应商编码 |
| sku_code | sku_code | SKU编码 |
| qty_expected | qty_expected | 预期数量 |
| eta_date | eta_date | 预计到货日期 |

---

### Story 5.2 入库结果回传
**作为** ERP系统  
**我希望** 能够接收WMS回传的入库结果  
**以便** 更新库存账和生成应付账款

**验收标准:**
- [ ] 入库完成后自动回传ERP
- [ ] 回传字段包括：PO号、实际入库数量、差异数量、入库时间
- [ ] 支持分批入库回传
- [ ] 回传失败时重试3次，仍失败则人工处理
- [ ] ERP确认接收后更新回传状态

---

### Story 5.3 采购订单状态同步
**作为** 采购员  
**我希望** 能够在ERP查看采购订单的WMS执行状态  
**以便** 跟踪采购进度

**验收标准:**
- [ ] WMS入库单状态变更时同步到ERP
- [ ] 同步状态：待收货/收货中/验收中/待上架/已完成
- [ ] 可以查看各环节处理时间
- [ ] 异常状态（取消/差异）同步通知

## 三、库存同步

### Story 5.7 库存实时同步到ERP
**作为** ERP系统  
**我希望** 能够实时获取WMS库存数据  
**以便** 保持库存数据一致

**验收标准:**
- [ ] 库存变动时实时同步到ERP
- [ ] 同步字段：SKU、仓库、库位、数量、批次
- [ ] 支持全量同步（定时任务）
- [ ] 支持增量同步（实时推送）
- [ ] 同步失败时记录日志并告警

---

### Story 5.8 库存同步到电商平台
**作为** 电商平台卖家  
**我希望** 库存变动时自动同步到各平台  
**以便** 防止超卖

**验收标准:**
- [ ] 库存变动时自动计算可售库存
- [ ] 可售库存 = 可用库存 - 平台预留
- [ ] 自动同步到Shopee/Lazada/TikTok
- [ ] 支持设置各平台库存比例
- [ ] 同步失败时标记并支持重试

**可售库存计算:**
```
可售库存 = 可用库存 - 锁定库存 - 安全库存
平台分配库存 = 可售库存 × 平台比例
```
