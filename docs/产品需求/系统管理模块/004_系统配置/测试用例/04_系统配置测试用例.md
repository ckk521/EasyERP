# 系统配置测试用例

## 一、配置项列表

### TC-CONF-001: 配置项列表-默认展示
**前置条件**：系统已初始化配置
**操作步骤**：
1. 进入系统配置页面
**预期结果**：
- 显示所有配置项列表
- 列表包含：配置编码、配置名称、配置值、配置类型、更新时间、操作

### TC-CONF-002: 配置项分类展示
**前置条件**：系统存在多类配置项
**操作步骤**：
1. 选择分类筛选：单据配置
**预期结果**：
- 仅显示单据相关的配置项：
  - 入库单号前缀
  - 出库单号前缀
  - 入库单号规则

### TC-CONF-003: 配置项搜索
**前置条件**：存在多个配置项
**操作步骤**：
1. 在搜索框输入"入库"
2. 点击搜索
**预期结果**：
- 返回所有配置名称或编码包含"入库"的配置项

---

## 二、单据编号配置

### TC-CONF-010: 修改入库单号前缀
**前置条件**：当前入库单号前缀为"RK"
**操作步骤**：
1. 找到"入库单号前缀"配置项
2. 点击"编辑"按钮
3. 将前缀改为"IN"
4. 点击保存
**预期结果**：
- 配置更新成功
- 提示"配置已保存"
- 新建的入库单号将使用新前缀：IN20260410001

### TC-CONF-011: 修改出库单号前缀
**前置条件**：当前出库单号前缀为"CK"
**操作步骤**：
1. 找到"出库单号前缀"配置项
2. 点击"编辑"
3. 将前缀改为"OUT"
4. 点击保存
**预期结果**：
- 配置更新成功
- 新建的出库单号将使用新前缀：OUT20260410001

### TC-CONF-012: 修改盘点单号前缀
**前置条件**：当前盘点单号前缀为"PD"
**操作步骤**：
1. 找到"盘点单号前缀"配置项
2. 点击"编辑"
3. 将前缀改为"CKPD"
4. 点击保存
**预期结果**：
- 配置更新成功

### TC-CONF-013: 单号前缀格式校验
**前置条件**：有配置编辑权限
**操作步骤**：
1. 编辑入库单号前缀
2. 输入格式：123（含数字）
3. 点击保存
**预期结果**：
- 提示"单号前缀只能为字母"
- 保存被拒绝

### TC-CONF-014: 单号前缀长度校验
**前置条件**：有配置编辑权限
**操作步骤**：
1. 编辑入库单号前缀
2. 输入超过5个字符的字符串
3. 点击保存
**预期结果**：
- 提示"单号前缀长度不能超过5个字符"
- 保存被拒绝

## 三、预警配置

### TC-CONF-020: 修改盘点差异阈值
**前置条件**：当前盘点差异阈值为5%
**操作步骤**：
1. 找到"盘点差异阈值"配置项
2. 点击"编辑"
3. 将阈值改为10%
4. 点击保存
**预期结果**：
- 配置更新成功
- 盘点差异在10%以内的自动审核通过
- 超过10%的需要人工审核

### TC-CONF-021: 修改效期预警天数
**前置条件**：当前效期预警天数为30天
**操作步骤**：
1. 找到"效期预警天数"配置项
2. 点击"编辑"
3. 将天数改为15
4. 点击保存
**预期结果**：
- 配置更新成功
- 商品过期前15天开始发送预警通知

### TC-CONF-022: 修改拣货超时提醒
**前置条件**：当前拣货超时提醒为30分钟
**操作步骤**：
1. 找到"拣货超时提醒"配置项
2. 点击"编辑"
3. 将时间改为60分钟
4. 点击保存
**预期结果**：
- 配置更新成功
- 拣货超过60分钟未完成发送超时提醒

### TC-CONF-023: 预警数值范围校验
**前置条件**：有配置编辑权限
**操作步骤**：
1. 编辑效期预警天数
2. 输入负数：-10
3. 点击保存
**预期结果**：
- 提示"预警天数必须为正整数"
- 保存被拒绝

### TC-CONF-024: 预警数值范围校验-超过上限
**前置条件**：有配置编辑权限
**操作步骤**：
1. 编辑效期预警天数
2. 输入超过365的数值
3. 点击保存

## 八、配置导入导出

### TC-CONF-070: 导出配置
**前置条件**：存在配置数据
**操作步骤**：
1. 点击"导出配置"按钮
2. 选择导出范围：全部/当前分类
3. 确认导出
**预期结果**：
- 生成包含所有配置项的JSON或Excel文件
- 文件自动下载

### TC-CONF-071: 导入配置
**前置条件**：有配置导入文件
**操作步骤**：
1. 点击"导入配置"按钮
2. 选择导入文件
3. 确认导入
**预期结果**：
- 配置导入成功
- 提示"成功导入X项配置"
- 或提示导入失败的配置项及原因

### TC-CONF-072: 导入配置-格式错误
**前置条件**：有格式错误的导入文件
**操作步骤**：
1. 选择导入文件
2. 点击确认导入
**预期结果**：
- 提示"导入文件格式错误"
- 或提示具体哪一行数据有问题

### TC-CONF-073: 导入配置-覆盖确认
**前置条件**：导入文件包含已存在的配置
**操作步骤**：
1. 选择导入文件（包含已存在配置）
2. 点击确认导入
**预期结果**：
- 提示"以下配置已存在，是否覆盖？"
- 显示将被覆盖的配置列表
- 用户确认后执行覆盖
