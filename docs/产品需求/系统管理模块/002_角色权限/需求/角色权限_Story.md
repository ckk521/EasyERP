# 角色权限管理 - 用户故事

> 模块说明: 角色管理、权限配置、用户角色分配
> 优先级: P0 (MVP必需)

---

## Story 2.1 角色管理（MVP简化版）
**作为** 系统管理员
**我希望** 能够管理系统角色
**以便** 按角色分配权限

**验收标准:**
- [ ] 预置3个基础角色：
  - **系统管理员**：全部权限
  - **仓库管理员**：本仓库全部业务权限
  - **操作员**：入库/出库/库存操作权限（不含系统配置）
- [ ] 超级管理员（系统内置，不可删除）
- [ ] 角色列表显示角色名称、编码、描述
- [ ] MVP版本不支持自定义创建角色（Phase 2支持）

**MVP预置角色权限矩阵：**

| 功能模块 | 系统管理员 | 仓库管理员 | 操作员 |
|---------|-----------|-----------|--------|
| 系统管理 | ✅ 全部 | ❌ | ❌ |
| 用户管理 | ✅ 全部 | ❌ | ❌ |
| 仓库管理 | ✅ 全部 | ✅ 全部 | ✅ 查看 |
| 基础数据 | ✅ 全部 | ✅ 全部 | ✅ 查看 |
| 入库管理 | ✅ 全部 | ✅ 全部 | ✅ 操作 |
| 出库管理 | ✅ 全部 | ✅ 全部 | ✅ 操作 |
| 库存管理 | ✅ 全部 | ✅ 全部 | ✅ 查看+盘点 |
| 报表管理 | ✅ 全部 | ✅ 全部 | ✅ 查看 |
| 数据权限范围 | 全部仓库 | 本仓库 | 本仓库 |

**数据权限：**
- 系统管理员：可操作所有仓库数据
- 仓库管理员：只能操作所属仓库的数据
- 操作员：只能操作所属仓库的数据

## Story 2.2 权限配置（按钮级）
**作为** 系统管理员
**我希望** 能够配置角色按钮权限
**以便** 控制角色的操作范围

**验收标准:**
- [ ] 支持按钮级权限控制：
  - 查看（列表页、详情页）
  - 新增
  - 编辑
  - 删除
  - 导出
  - 审核/审批
- [ ] 权限修改即时生效
- [ ] 按钮根据权限显示/隐藏

**权限标识规范：**
```
{模块}:{实体}:{操作}
示例：
  system:user:view     - 用户查看
  system:user:create   - 用户创建
  system:user:edit    - 用户编辑
  system:user:delete  - 用户删除
  inbound:order:view   - 入库单查看
  inbound:order:create - 入库单新增
```
