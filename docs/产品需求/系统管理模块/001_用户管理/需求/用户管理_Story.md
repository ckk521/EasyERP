# 用户管理 - 用户故事

> 模块说明: 用户账号管理、认证机制、会话管理、个人设置
> 优先级: P0 (MVP必需)

---

## Story 1.1 用户账号管理
**作为** 系统管理员
**我希望** 能够管理系统用户账号
**以便** 控制谁能访问系统

**验收标准:**
- [ ] 可以创建用户账号
- [ ] 填写基本信息：用户名、姓名、邮箱、手机号
- [ ] 设置初始密码（默认密码：111111）
- [ ] 可以编辑用户信息
- [ ] 可以重置密码
- [ ] 可以设置用户状态：正常/禁用/离职
- [ ] 可以删除账号（软删除）
- [ ] 列表显示所有用户及状态

**用户状态说明：**
| 状态 | 说明 | 能否登录 | 能否操作 |
|------|------|---------|---------|
| 正常 | 正常使用 | ✅ | ✅ |
| 禁用 | 账号被禁用 | ❌ | ❌ |
| 离职 | 员工离职 | ❌ | ❌（历史数据保留）|

## Story 1.2 认证机制（JWT）
**作为** 系统
**我希望** 能够实现JWT Token认证
**以便** 安全地验证用户身份

**验收标准:**
- [ ] 使用JWT Token进行身份认证
- [ ] Access Token有效期1小时
- [ ] Refresh Token有效期7天
- [ ] 支持登录/登出
- [ ] Token中包含用户ID、用户名、角色信息

---

## Story 1.3 会话管理
**作为** 系统管理员
**我希望** 能够管理用户会话
**以便** 控制用户登录状态

**验收标准:**
- [ ] 可配置会话超时时间（默认30分钟无操作自动登出）
- [ ] 记录用户最后登录时间和IP地址
- [ ] 管理员可查看当前在线用户
- [ ] 管理员可强制下线指定用户
- [ ] 支持同一账号多设备登录（可配置）

---

## Story 1.4 敏感操作二次验证
**作为** 系统管理员
**我希望** 对敏感操作进行二次验证
**以便** 防止误操作或恶意操作

**验收标准:**
- [ ] 删除用户需要二次确认弹窗
- [ ] 删除角色需要二次确认弹窗
- [ ] 批量操作需要二次确认弹窗
- [ ] 确认时显示操作摘要

**二次验证场景：**
| 操作类型 | 验证方式 | 说明 |
|---------|---------|------|
| 删除用户 | 弹窗确认 | 显示用户名和关联角色 |
| 删除角色 | 弹窗确认 | 显示角色名称和关联用户数 |
| 批量删除 | 弹窗确认 | 显示将要删除的数据条数 |
| 批量审核 | 弹窗确认 | 显示将要审核的数据摘要 |

---

## Story 1.5 API权限控制
**作为** 系统
**我希望** 对后端API进行权限校验
**以便** 防止未授权的API调用

**验收标准:**
- [ ] 每个接口标注需要的权限标识（如 `system:user:create`）
- [ ] 网关/拦截器统一校验Token + 权限
- [ ] 无权限时返回403 Forbidden
- [ ] Token无效或过期时返回401 Unauthorized

## Story 1.6 用户个人设置
**作为** 普通用户
**我希望** 能够修改个人设置
**以便** 管理自己的账号信息

**验收标准:**
- [ ] 可以修改个人资料（姓名、邮箱、手机号）
- [ ] 可以修改密码（需验证原密码）
- [ ] 可以设置头像
- [ ] 可以设置语言偏好（中文/英文）
- [ ] 可以查看登录历史（最近10条）
