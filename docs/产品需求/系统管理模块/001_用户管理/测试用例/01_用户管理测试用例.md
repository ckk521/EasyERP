# 用户管理测试用例

## 一、用户列表查询

### TC-USER-001: 查询用户列表-默认分页
**前置条件**：系统存在多个用户
**操作步骤**：
1. 进入用户管理页面
2. 点击查询按钮（不输入任何筛选条件）
**预期结果**：
- 返回用户列表，默认每页20条
- 显示用户总数
- 列表包含：用户名、姓名、手机号、邮箱、角色、所属仓库、状态、创建时间

### TC-USER-002: 查询用户列表-按用户名搜索
**前置条件**：系统存在用户 user_admin
**操作步骤**：
1. 在搜索框输入"user_admin"
2. 点击查询
**预期结果**：
- 返回匹配的用户列表
- 仅显示包含"user_admin"的用户

### TC-USER-003: 查询用户列表-按角色筛选
**前置条件**：系统存在多个角色的用户
**操作步骤**：
1. 选择角色下拉框，选择"仓库管理员"
2. 点击查询
**预期结果**：
- 仅返回角色为"仓库管理员"的用户

### TC-USER-004: 查询用户列表-按状态筛选
**前置条件**：系统同时存在启用和停用用户
**操作步骤**：
1. 选择状态下拉框，选择"停用"
2. 点击查询
**预期结果**：
- 仅返回状态为"停用"的用户

### TC-USER-005: 查询用户列表-按仓库筛选
**前置条件**：用户关联不同仓库
**操作步骤**：
1. 选择仓库下拉框，选择"越南胡志明仓"
2. 点击查询
**预期结果**：
- 仅返回所属仓库为"越南胡志明仓"的用户

## 二、新增用户

### TC-USER-010: 新增用户-必填字段验证
**前置条件**：登录账号有用户管理新增权限
**操作步骤**：
1. 点击"新增用户"按钮
2. 不填写任何字段，直接点击保存
**预期结果**：
- 提示"用户名不能为空"
- 提示"姓名不能为空"
- 提示"角色不能为空"
- 保存按钮保持可用，不关闭弹窗

### TC-USER-011: 新增用户-成功创建
**前置条件**：登录账号有用户管理新增权限
**操作步骤**：
1. 点击"新增用户"按钮
2. 填写用户信息：
   - 用户名：test_user_01
   - 姓名：张三
   - 手机号：13800138000
   - 邮箱：zhangsan@example.com
   - 角色：仓库管理员
   - 所属仓库：越南胡志明仓
3. 点击保存
**预期结果**：
- 用户创建成功
- 提示"用户创建成功"
- 用户列表刷新，新用户显示在列表中
- 用户状态默认为"启用"

### TC-USER-012: 新增用户-用户名重复校验
**前置条件**：系统已存在用户名为"test_user_01"的用户
**操作步骤**：
1. 点击"新增用户"按钮
2. 用户名填写"test_user_01"
3. 姓名填写：李四
4. 角色选择：入库员
5. 点击保存
**预期结果**：
- 提示"用户名已存在"
- 不关闭新增弹窗

### TC-USER-013: 新增用户-手机号格式校验
**前置条件**：登录账号有用户管理新增权限
**操作步骤**：
1. 点击"新增用户"按钮
2. 填写用户信息：
   - 用户名：test_user_02
   - 手机号：12345（格式错误）
3. 点击保存
**预期结果**：
- 提示"手机号格式不正确"

### TC-USER-014: 新增用户-邮箱格式校验
**前置条件**：登录账号有用户管理新增权限
**操作步骤**：
1. 点击"新增用户"按钮
2. 填写用户信息：
   - 用户名：test_user_03
   - 邮箱：invalid-email
3. 点击保存
**预期结果**：
- 提示"邮箱格式不正确"

## 三、编辑用户

### TC-USER-020: 编辑用户-修改基本信息
**前置条件**：存在用户 test_user_01
**操作步骤**：
1. 在用户列表找到 test_user_01
2. 点击"编辑"按钮
3. 修改姓名改为"张三修改"
4. 修改手机号为13900139000
5. 点击保存
**预期结果**：
- 用户信息更新成功
- 提示"用户信息已更新"
- 列表中显示更新后的信息

### TC-USER-021: 编辑用户-修改角色
**前置条件**：存在用户 test_user_01，角色为仓库管理员
**操作步骤**：
1. 点击用户"编辑"按钮
2. 将角色从"仓库管理员"改为"入库员"
3. 点击保存
**预期结果**：
- 角色更新成功
- 用户权限相应变化

### TC-USER-022: 编辑用户-修改所属仓库
**前置条件**：存在用户 test_user_01，关联越南仓
**操作步骤**：
1. 点击用户"编辑"按钮
2. 将所属仓库从"越南胡志明仓"改为"深圳总仓"
3. 点击保存
**预期结果**：
- 所属仓库更新成功
- 用户只能操作新仓库的数据

## 五、删除用户

### TC-USER-040: 删除用户-确认删除
**前置条件**：存在用户 test_user_del（无关联业务数据）
**操作步骤**：
1. 在用户列表找到 test_user_del
2. 点击"删除"按钮
3. 在确认弹窗中点击"确定"
**预期结果**：
- 用户被删除
- 提示"用户已删除"
- 用户从列表中消失

### TC-USER-041: 删除用户-取消删除
**前置条件**：存在用户 test_user_del
**操作步骤**：
1. 点击"删除"按钮
2. 在确认弹窗中点击"取消"
**预期结果**：
- 用户不被删除
- 弹窗关闭
- 用户仍在列表中

### TC-USER-042: 删除用户-有业务数据关联
**前置条件**：用户有关联的入库/出库操作记录
**操作步骤**：
1. 尝试删除该用户
**预期结果**：
- 提示"该用户存在业务数据关联，无法删除"
- 删除操作被拒绝

## 八、权限验证

### TC-USER-070: 无权限用户-访问用户管理
**前置条件**：当前用户角色无"用户管理"权限
**操作步骤**：
1. 尝试访问用户管理页面
**预期结果**：
- 提示"您没有访问该模块的权限"
- 页面跳转到首页或显示无权限提示

### TC-USER-071: 有查看权限-无编辑权限
**前置条件**：当前用户角色只有用户查看权限
**操作步骤**：
1. 进入用户管理页面
2. 尝试点击"新增"按钮
**预期结果**：
- 新增按钮不可见或禁用
- 或提示"您没有新增用户的权限"

### TC-USER-072: 仓库管理员-只能操作本仓库用户
**前置条件**：仓库管理员用户，操作员属于不同仓库
**操作步骤**：
1. 以仓库管理员身份登录
2. 查看用户列表
**预期结果**：
- 仅显示本仓库的用户
- 无法看到其他仓库的用户信息
