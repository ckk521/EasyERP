# PDA移动端（H5）详细设计文档

> 文档说明: 为仓库作业人员提供的移动端H5解决方案，支持扫码作业
> 适用设备: 工业PDA、手机、平板（建议PDA为主）
> 版本: 1.0 | 日期: 2026-04-09

---

## 一、整体架构设计

### 1.1 技术选型

| 层级 | 技术方案 | 说明 |
|------|---------|------|
| 前端框架 | Vue3 + Vant4 | 轻量级，适合移动端，组件丰富 |
| 状态管理 | Pinia | 简洁，适合中小型应用 |
| 路由 | Vue Router 4 | 支持动态路由 |
| HTTP请求 | Axios | 统一拦截处理 |
| 扫码 | 设备原生扫码 + 摄像头备用 | 优先PDA硬件扫码 |
| 离线存储 | IndexedDB | 支持弱网/断网作业 |
| PWA | Workbox | 可添加到主屏幕，接近原生体验 |

### 1.2 部署方式

```
┌─────────────────────────────────────────┐
│           PDA设备（H5页面）              │
│  ┌─────────────────────────────────┐   │
│  │  方式1: 浏览器访问               │   │
│  │  https://wms.company.com/pda    │   │
│  └─────────────────────────────────┘   │
│  ┌─────────────────────────────────┐   │
│  │  方式2: PWA添加到主屏幕          │   │
│  │  像App一样使用，全屏体验         │   │
│  └─────────────────────────────────┘   │
│  ┌─────────────────────────────────┐   │
│  │  方式3: WebView嵌入（可选）      │   │
│  │  原生App壳包裹H5                 │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

### 1.3 与PC端的关系

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   PC管理端   │◄────►│   后端API    │◄────►│   PDA作业端  │
│  (Vue/React)│      │  (RESTful)  │      │   (H5)      │
└─────────────┘      └─────────────┘      └─────────────┘
      │                                          │
      │  管理功能：                               │  作业功能：
      │  - 单据创建/分配                          │  - 扫码作业
      │  - 数据查看/报表                          │  - 任务领取
      │  - 系统配置                               │  - 结果提交
      └──────────────────────────────────────────┘
                        数据同步
```

## 二、核心设计原则

### 2.1 交互设计原则

| 原则 | 说明 | 具体措施 |
|------|------|----------|
| **单手操作** | 员工可能一手拿货物 | 主要按钮在屏幕下方，支持手势 |
| **大按钮** | 戴手套操作 | 按钮最小60x60px，间距充足 |
| **强反馈** | 嘈杂环境听不到提示音 | 震动+视觉+声音三重反馈 |
| **防误触** | 避免重复提交 | 操作后按钮禁用，显示加载状态 |
| **离线可用** | 仓库信号可能不好 | 关键数据本地缓存，支持离线作业 |

### 2.2 扫码设计原则

```
扫码输入优先级：
1. PDA硬件扫码枪（最快，推荐）
2. 手机摄像头扫码（备用）
3. 手动输入（兜底）

扫码后自动处理流程：
扫描条码 → 解析内容 → 自动匹配 → 填充字段 → 焦点跳转
```

### 2.3 页面布局规范

```
┌─────────────────────────────┐  ◄── 状态栏（固定）
│ 时间  电量  网络  登录用户    │
├─────────────────────────────┤  ◄── 标题栏（固定）
│  <  页面标题          [刷新] │
├─────────────────────────────┤  ◄── 任务信息区
│  当前任务：入库单 RK001      │
│  商品：3/10  已完成          │
├─────────────────────────────┤  ◄── 主内容区（可滚动）
│                             │
│      扫码/输入区域           │
│                             │
│      商品信息显示            │
│                             │
│      操作按钮                │
│                             │
├─────────────────────────────┤  ◄── 底部操作栏（固定）
│  [暂存]    [提交]   [跳过]  │
└─────────────────────────────┘
```

## 三、功能模块详细设计

### 3.1 登录模块

#### Story PDA-1: PDA端登录
**作为** 仓库作业员  
**我希望** 能够在PDA上登录系统  
**以便** 接收和完成作业任务

**详细设计:**

```
登录页面布局：
┌─────────────────────────────┐
│                             │
│        [公司Logo]           │
│                             │
│    ┌─────────────────┐     │
│    │  账号/工号       │     │
│    └─────────────────┘     │
│                             │
│    ┌─────────────────┐     │
│    │  密码           │     │
│    └─────────────────┘     │
│                             │
│    ┌─────────────────┐     │
│    │  选择仓库  ▼    │     │
│    └─────────────────┘     │
│                             │
│    [ ] 记住密码             │
│                             │
│    ┌─────────────────┐     │
│    │     登 录       │     │
│    └─────────────────┘     │
│                             │
└─────────────────────────────┘
```

**验收标准:**
- [ ] 支持账号/工号+密码登录
- [ ] 必须选择仓库（影响数据范围）
- [ ] 支持记住密码（本地存储，加密）
- [ ] 登录失败3次后需要验证码
- [ ] 登录成功后缓存Token（7天有效）
- [ ] 网络异常时提示"网络连接失败"
- [ ] 支持扫码快速登录（高级功能）

**边界条件:**
| 场景 | 处理方式 |
|------|----------|
| 网络断开 | 提示"网络异常，请检查网络"，提供离线模式入口 |
| 密码错误3次 | 显示图形验证码，第5次锁定账号15分钟 |
| Token过期 | 自动跳转登录页，保留当前页面路径，登录后返回 |
| 多设备登录 | 允许，但记录登录设备，可后台强制下线 |
| 账号被禁用 | 提示"账号已禁用，请联系管理员" |

### 3.3 入库作业模块

#### Story PDA-4: 收货作业
**作为** 收货员  
**我希望** 能够扫描商品进行收货  
**以便** 快速确认到货数量

**详细设计:**

```
│ 进度: 5/10  已完成50%       │
│ [==========>        ]       │
├─────────────────────────────┤
│                             │
│  ┌─────────────────────┐   │
│  │  📷 点击扫码或输入  │   │  ◄── 扫码区域（大按钮）
│  └─────────────────────┘   │
│                             │
│  或手动输入：               │
│  ┌─────────────────────┐   │
│  │ SKU/条码            │   │
│  └─────────────────────┘   │
│                             │
├─────────────────────────────┤
│  已扫描商品：               │
│  ┌─────────────────────┐   │
│  │ 📦 iPhone15手机壳   │   │
│  │ 条码：123456789     │   │
│  │ 应收：100  实收：100│   │
│  │ [+] [-]  [删除]     │   │
│  └─────────────────────┘   │
│                             │
├─────────────────────────────┤
│ [暂存]  [完成收货]  [异常]  │
└─────────────────────────────┘
```

**作业流程:**
```
开始收货
   │
   ▼
扫描商品条码 ──► 系统自动识别SKU
   │
   ▼
显示商品信息（名称、规格、图片）
   │
   ▼
输入/确认收货数量（默认1）
   │
   ▼
添加到收货列表
   │
   ▼
继续扫描 或 完成收货
```

**验收标准:**
- [ ] 扫描条码自动识别商品
- [ ] 显示商品主图、名称、规格
- [ ] 默认收货数量1，可修改
- [ ] 支持连续扫码（扫一个自动下一个）
- [ ] 已收商品列表可查看、修改、删除
- [ ] 支持部分收货（分批到货）
- [ ] 收货完成提交后生成验收任务

**边界条件:**
| 场景 | 处理方式 |
|------|----------|
| 条码不存在 | 提示"商品不存在"，提供手动选择入口 |
| 条码模糊无法识别 | 提示"识别失败，请调整光线重试"，提供手动输入 |
| 收货数量>预期数量 | 提示"超收，请确认"，需主管密码确认 |
| 收货数量<预期数量 | 标记为部分收货，记录差异 |
| 商品破损 | 点击"异常"按钮，拍照+选择原因 |
| 网络中断 | 本地缓存，恢复后自动同步 |
| PDA没电 | 自动保存草稿，换设备登录可继续 |

#### Story PDA-11: 离线模式
**作为** 仓库作业员  
**我希望** 网络不好时也能继续作业  
**以便** 不影响工作效率

**设计方案:**

```
离线数据存储结构：
┌─────────────────────────────────────┐
│ IndexedDB: WMS_PDA                  │
├─────────────────────────────────────┤
│ 表: pending_tasks                   │
│  - 待同步的任务数据                  │
├─────────────────────────────────────┤
│ 表: task_cache                      │
│  - 已领取的任务详情                  │
├─────────────────────────────────────┤
│ 表: product_cache                   │
│  - 商品基础数据缓存                  │
├─────────────────────────────────────┤
│ 表: location_cache                  │
│  - 库位数据缓存                      │
├─────────────────────────────────────┤
│ 表: operation_log                   │
│  - 操作日志（用于审计）              │
└─────────────────────────────────────┘
```

**验收标准:**
- [ ] 网络断开时自动切换离线模式
- [ ] 离线模式下可继续作业（收货/上架/拣货等）
- [ ] 数据本地存储（IndexedDB）
- [ ] 网络恢复后自动同步数据
- [ ] 同步冲突时提示人工处理

**边界条件:**
| 场景 | 处理方式 |
|------|----------|
| 离线时领取新任务 | 提示"网络异常，无法领取新任务" |
| 离线时提交任务 | 本地保存，提示"已保存，联网后自动同步" |
| 同步时冲突（他人已修改） | 提示"数据冲突，请人工确认"，展示差异 |
| 长时间离线（>1天） | 提示"数据已过期，请重新登录同步" |

|------|------|------|
| /pda/auth/login | POST | PDA登录 |
| /pda/tasks | GET | 获取任务列表 |
| /pda/tasks/{id}/claim | POST | 领取任务 |
| /pda/tasks/{id}/release | POST | 释放任务 |
| /pda/inbound/receive | POST | 收货提交 |
| /pda/inbound/inspect | POST | 验收提交 |
| /pda/inbound/putaway | POST | 上架提交 |
| /pda/outbound/pick | POST | 拣货提交 |
| /pda/outbound/pack | POST | 打包提交 |
| /pda/inventory/count | POST | 盘点提交 |
| /pda/sync | POST | 离线数据同步 |
| /pda/product/{barcode} | GET | 扫码查询商品 |
| /pda/location/{code} | GET | 扫码查询库位 |

### 4.2 接口规范

**统一响应格式:**
```json
{
  "code": 200,
  "message": "success",
  "data": { }
}
```

**错误码定义:**
| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| 200 | 成功 | 正常处理 |
| 400 | 参数错误 | 提示用户检查输入 |
| 401 | 未登录/Token过期 | 跳转登录页 |
| 403 | 无权限 | 提示"无权操作" |
| 404 | 资源不存在 | 提示"数据不存在" |
| 409 | 数据冲突 | 提示"数据已被修改，请刷新" |
| 500 | 服务器错误 | 提示"服务器异常，请重试" |
| 503 | 网络异常 | 切换离线模式 |

## 五、性能优化

### 5.1 加载优化
- 路由懒加载，按需加载页面
- 图片懒加载，使用占位图
- 数据分页，避免一次性加载过多
- 本地缓存商品/库位基础数据

### 5.2 渲染优化
- 使用v-show替代v-if（频繁切换场景）
- 长列表使用虚拟滚动
- 避免不必要的计算属性

### 5.3 网络优化
- API请求防抖/节流
- 批量提交（离线数据同步）
- 请求失败自动重试（3次）
- 超时设置：10秒
